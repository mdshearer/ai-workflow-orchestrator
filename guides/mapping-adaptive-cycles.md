### Step 4: Identify Phase Transitions (10 minutes)

Phase transitions happen when your **mindset fundamentally shifts**.

#### râ†’K Transition: From Exploration to Production

**When does this happen?**
- Strategy is approved
- Research is complete
- Requirements are clear
- Plan is locked in

**Examples:**
- **Content:** Content brief approved â†’ Start outlining
- **Software:** Requirements finalized â†’ Start architecture
- **Design:** User research complete â†’ Start sketching

**What changes:**
- Uncertainty â†’ Clarity
- Many options â†’ One path
- Divergent â†’ Convergent

---

#### Kâ†’Î© Transition: From Production to Critique

**When does this happen?**
- Work product is complete
- Ready for evaluation
- Time for quality check
- Submission for review

**Examples:**
- **Content:** Draft complete â†’ Critical review
- **Software:** Code complete â†’ Code review
- **Design:** Prototype complete â†’ Usability testing

**What changes:**
- Builder mindset â†’ Judge mindset
- Friendly â†’ Adversarial
- Creating â†’ Evaluating

---

#### Î©â†’Î± Transition: From Critique to Evolution (CONDITIONAL)

**When does this happen?**
- Critical failure detected
- Score below threshold (e.g., <50/100)
- Recurring pattern (3rd occurrence of same issue)
- Systemic problem identified

**Examples:**
- **Content:** Blog post scores 42/100 â†’ Root cause analysis
- **Software:** Security audit fails â†’ Update security standards
- **Design:** Usability test fails badly â†’ Revise design system

**What changes:**
- "What's wrong with THIS?" â†’ "What's wrong with OUR SYSTEM?"
- Tactical fixes â†’ Strategic improvements
- Product-focused â†’ Process-focused

**IMPORTANT:** This transition is **conditional** - it doesn't happen on every project, only when significant failures occur.

---

#### Î±â†’r Transition: From Evolution to Next Cycle

**When does this happen?**
- System learnings captured
- Constitution/standards updated
- Root causes addressed
- Next cycle strategy defined

**Examples:**
- **Content:** Style guide updated â†’ Next article uses new rules
- **Software:** Coding standards improved â†’ Next feature follows them
- **Design:** Design patterns documented â†’ Next project benefits

**What changes:**
- System is smarter
- Recurring mistakes prevented
- Process evolved

**Slow, predictable, capital-building**

1. **r-phase:** Explore and gather (strategy, research, planning)
2. **K-phase:** Build and refine (production, optimization)

### Back Loop (Î©â†’Î±): 10-20% of time
