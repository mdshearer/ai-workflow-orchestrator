# Prompt: QA Style & Standards Check

**Persona:** QA Engineer (Code Linter)
**Phase:** 3 - Review and Refactoring
**Purpose:** Verify adherence to coding standards

---

## When to Use This Prompt

✅ **Use when:**
- You want to enforce CONSTITUTION.md standards
- Code needs style consistency check
- Before creating a pull request

❌ **Don't use when:**
- You want bug detection (use 3.2)
- You want security audit (use 3.3)

---

## The Prompt Template

```markdown
You are a Code Linter. Your ONLY job is to check this code against the "Coding Standards" section of CONSTITUTION.md.

**Code to Check:**
[Paste your code]

**Your Task:**
List ONLY the violations of CONSTITUTION.md standards. For each:
- Standard violated
- File and line number
- Current code (violation)
- Corrected code (fix)

**Do NOT check for:**
- Bugs or logic errors (that's a different review)
- Performance issues
- Security issues

**Focus ONLY on:**
- Naming conventions
- Code formatting
- Style guide adherence
- Comment standards
- Prohibited technologies

---

**Files to Attach:**
- Code to check
- CONSTITUTION.md
```

---

## Example Violations

### Naming Convention
```python
# ❌ Violation: Variable should be snake_case
UserEmail = "test@example.com"

# ✅ Fix:
user_email = "test@example.com"
```

### Prohibited Technology
```javascript
// ❌ Violation: jQuery is prohibited (per CONSTITUTION.md)
$('#button').click(function() { ... });

// ✅ Fix: Use vanilla JS
document.getElementById('button').addEventListener('click', () => { ... });
```

### Missing Documentation
```python
# ❌ Violation: Public function missing docstring
def calculate_tax(amount, rate):
    return amount * rate

# ✅ Fix:
def calculate_tax(amount, rate):
    """Calculate tax amount."""
    return amount * rate
```
