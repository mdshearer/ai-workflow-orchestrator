# Prompt: README.md Generator

**Persona:** Technical Writer
**Phase:** 4 - Documentation
**Purpose:** Generate a comprehensive README.md for your project

---

## When to Use This Prompt

✅ **Use when:** Starting a new project, feature is complete, or updating project documentation
❌ **Don't use when:** Writing user guides for non-technical users (use 4.2)

---

## The Prompt Template

```markdown
You are a Technical Writer. Generate a comprehensive `README.md` file for this project.

**Rules:**
1. Review the entire project workspace
2. Follow standards from `CONSTITUTION.md`
3. Tone: Clear, concise, helpful for **developers**
4. Must be **actionable** (new developer can set up the project)

**Task:**
Create a `README.md` with these sections:

1. **Project Title & Description** (1-3 sentences: what it does)
2. **Key Features** (bullet list of main capabilities)
3. **Installation & Setup** (step-by-step prerequisites and install)
4. **Usage** (basic code example or how to run)
5. **Configuration** (env vars, config files - if applicable)
6. **API Documentation** (link to detailed docs, don't include full specs)
7. **Contributing** (how to contribute)
8. **License** (link to LICENSE file)
9. **Support/Contact** (who to ask for help)

**What NOT to Include:**
- Detailed API documentation (link to `docs/API.md`)
- Full license text (link to `LICENSE` file)
- Extensive troubleshooting (link to `docs/TROUBLESHOOTING.md`)
- Keep under 300 lines (concise!)

**Output:**
- Filename: `README.md`
- Save to project root

---

**Files to Read:**
- Project files (main code, package.json/requirements.txt, etc.)
- `CONSTITUTION.md`
```

---

## Quick Start

### Step 1: Invoke AI
The AI will:
- Scan the entire codebase
- Read CONSTITUTION.md
- Identify key features and setup steps

### Step 2: Review Generated README

Check:
- [ ] Description is clear
- [ ] Installation steps work
- [ ] Usage examples are helpful
- [ ] Links to detailed docs (not everything in README)
- [ ] Tone is professional

### Step 3: Save

```bash
git add README.md
git commit -m "docs: add/update README"
```

## Example README Structure

```markdown
# Project Name

One-sentence description of what this project does.

## Features

- User authentication with JWT
- Password reset via email
- Role-based access control

## Installation

### Prerequisites
- Node.js 18+
- PostgreSQL 15+

### Setup
\`\`\`bash
# Clone the repository
git clone https://github.com/yourorg/project.git
cd project

# Install dependencies
npm install

# Set up environment
cp .env.example .env
# Edit .env with your values

# Run database migrations
npm run migrate

# Start the server
npm run dev
\`\`\`

## Usage

\`\`\`typescript
import { authenticate } from './auth';

const token = await authenticate(email, password);
\`\`\`

## Configuration

Required environment variables:
- `DATABASE_URL` - PostgreSQL connection string
- `JWT_SECRET` - Secret for token signing

## API Documentation

See [API Documentation](./docs/API.md)

## Contributing

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## License

MIT License - see [LICENSE](./LICENSE)

## Support

Contact: dev-team@company.com
```

---

## Tips

✅ **DO:**
- Focus on getting started quickly
- Include real, working examples
- Link to detailed docs for deep dives
- Keep it under 300 lines

❌ **DON'T:**
- Include full API reference in README
- Paste entire license text
- Write essays (be concise!)
- Forget setup prerequisites
